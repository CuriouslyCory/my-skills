## Codebase Patterns
- shared-types vitest config: `vitest.config.ts` with `globals: true`, import `{ describe, it, expect }` from "vitest" explicitly in test files
- shared-types test location: `packages/shared-types/src/__tests__/`
- CLI vitest config: `apps/cli/vitest.config.ts` with `globals: true`
- API vitest config: `packages/api/vitest.config.ts` with `globals: true`
- CLI tests use real temp directories (mkdtemp + rm cleanup)
- pnpm filter names: `my-skills` (CLI), `@curiouslycory/shared-types`, `@curiouslycory/api`, `@curiouslycory/web`
- `sourceToGitHub` was duplicated in add.ts, check.ts, update.ts â€” extracted to `services/source-parser.ts` for testability
- Mocking `@curiouslycory/git-service`: use `vi.hoisted()` + class-based mock (class with property assignments) â€” `vi.fn().mockImplementation(() => ({...}))` does NOT work as constructor mock
- Use `() => undefined` not `() => {}` for mock implementations â€” `@typescript-eslint/no-empty-function` rejects empty arrow functions
- `vi.restoreAllMocks()` clears `mockReturnValue` set during `vi.hoisted()` â€” re-apply chainable returns (e.g., ora spinner) in `beforeEach`
- For `ora` mock: create spinner object with `start/succeed/fail/stop` fns, set `mockReturnValue(spinner)` on each â€” `mockReturnThis()` doesn't work on plain objects
- Avoid `importOriginal()` for manifest mock â€” inline pure functions (`addSkill`, `getSkill`) to avoid `@typescript-eslint/no-unsafe-return` lint errors
- Testing Commander commands: use `program.exitOverride()` + `program.parseAsync(["node", "ms", "<cmd>", ...args])` to exercise full flag parsing
- Inspecting mock call args: use `vi.mocked(fn).mock.calls[0][1] as unknown as Type` â€” avoids both `no-non-null-assertion` and `no-unsafe-assignment` lint rules
- Avoid nested `expect.objectContaining()` â€” inner matchers return `any`, triggering `no-unsafe-assignment`
- API router tests: mock `@curiouslycory/db/client` with `vi.mock("@curiouslycory/db/client", () => ({ db: {} }))` to prevent real SQLite init at import time
- API router tests: mock `../../lib/config-sync` to avoid filesystem writes from `syncConfigToFile`
- API test-utils: `createTestCaller` returns `{ db, caller, rawDb, repoPath }` â€” use `rawDb.exec("DELETE FROM ...")` to reset between tests
- SQLite `unixepoch()` defaults have 1-second granularity â€” don't assert ordering by timestamp when items are inserted in the same second
