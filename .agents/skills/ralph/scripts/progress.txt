## Codebase Patterns
- Web tests: Use `vi.hoisted(() => vi.fn())` for mocks that are referenced in `vi.mock()` factories — avoids temporal dead zone issues with hoisting
- Web tests: Import type-only via `import type * as Foo from "module"` for use inside `vi.mock(async (importOriginal) => ...)` to satisfy TS without unsafe-return
- Web tests: SkillList uses `useSuspenseQuery` from `@tanstack/react-query` — mock at the module level, not via tRPC mock alone
- Web tests: `next/link` must be mocked in web component tests (returns plain `<a>` tags)
- API tests: `createTestCaller` in test-utils.ts provides in-memory SQLite + drizzle + appRouter for router tests
- CLI tests: Use real temp directories (mkdtemp + rm cleanup) rather than mocking fs
