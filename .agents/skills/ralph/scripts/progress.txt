## Codebase Patterns
- drizzle-kit push: Use `pnpm --filter @curiouslycory/db push` (wraps `pnpm with-env drizzle-kit push`)
- drizzle-kit push will try to drop FTS virtual tables — apply schema changes via direct SQL if push hangs or errors on those tables
- DB location: `$DB_PATH` from .env, defaults to `./data/my-skills.db` relative to packages/db (resolves to repo root `data/my-skills.db`)
- Composite unique constraints in drizzle: use `unique()` import from `drizzle-orm/sqlite-core`, define in table's third argument as `(table) => [unique().on(table.col1, table.col2)]`
- SQLite NULL handling: NULL values are treated as distinct in unique constraints — add procedures must use explicit existence checks
- `isNull` from drizzle-orm is NOT re-exported from `drizzle-orm/sql` — added explicit re-export from `drizzle-orm/sql/expressions/conditions` in `packages/db/src/index.ts`
- When moving tRPC sub-routes (e.g., `config.favorites.*` → `favorite.*`), update all client references in web app components AND server prefetch calls
- zod/v4: `.default({})` on an object with inner `.default()` fields fails — outer default must include all required fields (e.g., `.default({ page: 1, pageSize: 30 })`)
- When changing a tRPC procedure return type, search for all consumers in the web app with `grep` and update them
- Server component prefetch pattern: `prefetch(trpc.X.queryOptions())` + `HydrateClient` wrapper + client `useSuspenseQuery(trpc.X.queryOptions())`
- Card imports from `@curiouslycory/ui/card`: `Card`, `CardHeader`, `CardTitle`, `CardDescription`, `CardContent`
- DataTable component at `apps/web/src/app/_components/data-table.tsx` — generic reusable table with sorting, filtering, pagination built on @tanstack/react-table + @curiouslycory/ui/table; supports optional `manualSorting`, `manualPagination`, `pageCount`, external `sorting`/`onSortingChange`/`pagination`/`onPaginationChange` props for server-side control
- URL state sync pattern: `useSearchParams` + `router.replace()` with `{ scroll: false }` — use `useMemo` for derived state to avoid lint warnings about unstable deps in `useCallback`
- ESLint requires top-level `import type` statements, not inline `type` specifiers (e.g., `import type { Foo } from "x"` not `import { type Foo } from "x"`)
- `useReactTable` triggers a `react-hooks/incompatible-library` warning — this is expected and matches the existing `fragment-selection-table.tsx` pattern
- chalk-colored strings break `String.padEnd()` due to ANSI escape codes — use fixed spacing for colored table columns
